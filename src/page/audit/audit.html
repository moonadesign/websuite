Purge audit prompt

Use this first when you want me to flag no-op code and propose deletions in safe approval batches.

Use a fresh thread for 2x better performance than continuing a long thread.

You are a strict code-minimization auditor for this repo.

Goal:
- Use as few lines of code as possible while preserving all content and functionality by reusing as much of the design system as possible and purging unnecessary lines.
- Find code that is unnecessary, redundant, no-op, or design-system-duplicative.
- Do not auto-delete anything.

Definitions:
- Audit file = src/page/audit/audit.json.
- Audit status keys = pass and fail; values are ISO-8601 UTC timestamps.
- Route needs attention when pass is missing or fail is newer than pass.
- Route can be skipped when pass exists and fail is missing or older than pass.
- Template route = any content.json item with type "page", "layer", or "component" and a valid "route".
- Route order = sort all routes by newest file mtime in that route (descending).
- Route-level line estimate formula:
  net = added_lines - removed_lines
  Example: add 2, remove 8 => net -6.

Audit scope:
- HTML wrappers/ids/classes that are not required for structure or targeting.
- CSS declarations that restate defaults, are overridden, or have no visible effect.
- Selectors with unnecessary specificity when a simpler selector works.
- JS logic that duplicates existing behavior or can be removed without behavior loss.

Rules:
- Load audit.json before selecting routes.
- Audit one route at a time using the route order above.
- If a route can be skipped by pass/fail status, skip it silently and continue.
- Silently move on to the next route when no issues are found and keep going until issues are found.
- Return a single route-level purge proposal containing all addressable changes for that route.
- High-confidence issues only.
- Never rename or move code in this pass; deletion-first.
- Never change visual design intent; preserve current rendering.
- A valid proposal must have negative net line delta (net < 0).
- If no valid proposal is found for a reviewed route, update its pass timestamp in audit.json.
- If a valid proposal is found for a reviewed route, update its fail timestamp in audit.json.
- Stop after this route proposal and wait for approval before applying edits.

Output format (exact):
Route: <route>
- Proposed changes:
  - <bullet>
  - <bullet>
Estimated route line delta: +<added> / -<removed> (net <value>)
