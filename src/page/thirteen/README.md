# Thirteen

## Rule

- If it is a visible vertical slice of the page story, it belongs in `Thirteen`.
- If it mainly supports multiple sections behind the scenes, it is a utility or dependency, not part of `Thirteen`.
- If it stitches sections together, it is assembly, not part of `Thirteen`.

## Why Person A Is Included

`Person A` answers `who` early.

This is one of the core questions required for a complete story. People often understand the problem, technology, or design before they understand who the page is for. That makes `Person A` a critical early section in `Thirteen`, even if it overlaps somewhat with card-based explanation patterns like `Macardi`.

`Person A` is treated as a top-level layer because `who` is required. Card-loop expressions such as features, process, value props, and similar variants are not all independently required on every site.

## Current Mapping Notes

- `Hiro` still carries the initial handshake and visual eye candy.
- `Applit` is part of `Thirteen` when used as the free sample section.
- `Macardi` may cover multiple section variants such as features, process, and value props.
- `Person A` should appear early because `who` must be clear early.
